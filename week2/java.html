자바란?

자바(Java)는 1995년 선 마이크로시스템즈(Sun Microsystems)에서 발표한 이후, 지금까지 전 세계적으로 가장 널리 쓰이는 객체 지향 프로그래밍 언어 중 하나이다. 웹 백엔드(대규모 금융권, 공공기관, 이커머스 서버 개발), 안드로이드 앱(구글의 안드로이드 OS용 애플리케이션 개발), 빅데이터 처리 프레임워크(하둡(Hadoop), 스파크(Spark) 등), 금융 및 엔터프라이즈(안정성이 중요한 대형 시스템 구축) 등의 다양한 분야에 널리 활용되고있다. 자바는 문법이 다소 엄격하고 다른 언어(Python 등)에 비해 코드량이 많을 수 있지만, 그만큼 정적 타입 체킹이 강력하여 런타임 에러를 줄일 수 있다는 장점이 있다. 안정적인 시스템 구축을 목표로 한다면 최고의 선택지 중 하나이다.

자바의 특징

1. 플랫폼 독립성

자바 컴파일러는 자바 소스 코드를 바이트코드로 변환하여 자바 가상 머신(JVM)에서 실행한다. JVM이 설치된 어떤 플랫폼에서도 동일한 바이트코드를 실행할 수 있기 때문에, 자바 프로그램은 다양한 운영체제와 하드웨어에서 호환성을 유지할 수 있다.(Write Once, Run Anywhere)

2. 객체 지향 프로그래밍

자바는 처음부터 객체 지향 언어로 설계되어, 캡슐화, 상속, 다형성 같은 OOP의 핵심 개념을 철저히 따른다. 모든 것을 객체 단위로 추상화하여 관리하므로 코드의 재사용성이 높고, 대규모 프로젝트에서 유지, 보수하기가 매우 유리하다.

3. 자동 메모리 관리

가비지 컬렉터(GC)가 더 이상 사용되지 않는 객체를 자동으로 찾아 메모리에서 제거해주기 때문에 C/C++와 달리 개발자가 직접 메모리를 해제할 필요가 없다.

4. 강력한 생태계와 프레임워크

Spring, JPA, Hibernate, JUnit 등 다양한 프레임워크를 통해 빠르고 안정적인 개발이 가능하다.
